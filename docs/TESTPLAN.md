# COBOL Student Account Test Plan

This test plan captures the current business logic and implementation behavior of the COBOL application so it can be validated with business stakeholders and reused during Node.js migration.

## Notes for Node.js Migration

- Preserve business rules first, then add enhancements as explicit new requirements.
- Keep operation code constants stable (`TOTAL `, `CREDIT`, `DEBIT `, `READ`, `WRITE`) in integration tests to verify parity.
- Convert TC-001 to TC-014 into black-box API/integration tests.
- Convert TC-015 to TC-018 into unit/integration tests around data-access and operation-routing modules.
